# IP 解释组图

把内容里的关键判断、机制、转折和隐喻，变成由同一 IP 角色亲自参与的 16:9 手绘解释图。

## 先确定家族与角色

在分析正文或生成图片前锁定一个主角色。

- 用户没有指定角色、也没有上传参考图：先问家族，给出 POP MART、Sanrio、Chiikawa、Peanuts、Moomin、Jellycat、MBTI/16Personalities、上传我的形象做 Q 版。
- 用户只指定家族：再列出该家族的快捷角色选项；按需读取 `references/ip-catalog.md`，不要一次倾倒所有家族的全部角色。
- 用户已明确指定角色或上传图片：不要重复询问，直接进入参考锁定。
- 用户只说“随便”：默认使用 TWINKLE TWINKLE。

第一问口径：

> 这组图想用哪个 IP 家族来讲解？可以选：POP MART、Sanrio、Chiikawa、Peanuts、Moomin、Jellycat、MBTI/16Personalities，也可以上传你的形象做 Q 版。

角色目录见 `references/ip-catalog.md`；角色卡与上传图规则见 `references/ip-modes.md`。

## 选择画风来源

按 `references/family-style-catalog.md` 使用三级优先级：

1. 角色或具体作品已有动画：使用该角色来源动画的视觉参考。
2. 角色没有独立动画，但家族有统一动画：使用家族动画画风参考。
3. 家族没有可确认的动画：使用 `references/visual-dna.md` 的默认简笔画。

Moomin、Peanuts 等家族存在多套动画风格时，优先匹配角色的原始作品或用户指定版本，不混用不同年代、2D、3D、真人或游戏渲染风格。画风名称只是检索键；精确匹配仍需用户有权使用的参考帧或已批准的生成画风板。

## 选择一致性等级

默认使用“参考锁定”，不要只靠模型记忆猜角色。

- 精确还原：要求用户提供 1-3 张该角色或具体系列的清晰参考图；没有参考图就暂停生图，只能继续做 shot list。
- 近似概念草图：允许只用官方英文名，但必须明确告知用户“身份可能漂移，不代表具体系列的精确形象”。
- 上传形象 Q 版：把上传图当身份参考，先建立角色锚点。

用户已经指出角色不像时，自动切换到“精确还原”，不要继续仅靠加长文字提示词重试。

## 查询私有参考图库

在要求用户重新上传前，先查询本机私有参考图库。按 `references/reference-library.md` 执行。

- 默认位置：`${CODEX_HOME}/ip-reference-library/`。若技能包附带 `assets/authorized-reference-library/`，先读取该目录，再按需同步到私有图库。
- 按“家族 / 官方英文角色名 / 具体系列或版本”查找，优先复用已通过检查的 1-3 张参考图。
- 本地缺图时，先请用户上传；用户允许联网检索时，只从权利方官方站点或官方商店寻找，并记录来源与版本。
- 不保存搜索引擎缩略图，不把未授权官方素材复制进可分享技能包，不把不同版本混成同一身份锚点。

## 建立双锚点

生成 3 张以上，或用户要求一致性时，必须建立并固定两个锚点。详细流程见 `references/consistency-pipeline.md`。

1. 身份锚点 `identity-master.png`：由用户提供的角色参考图生成；锁定轮廓、五官、比例、主色、服装和配饰。
2. 画风锚点 `style-master.png`：优先由对应动画参考帧生成；锁定线条、轮廓、色块、背景复杂度、光影、材质和构图气质。无动画时锁定默认简笔画 DNA。
3. 每张场景图都同时带入原始身份参考、身份锚点和画风锚点。不得只引用上一张成图，避免误差逐张累积。
4. 先生成一张测试场景并检查；不合格就修正锚点，不要批量生产错误身份。

每次任务只保留一个主 IP。角色必须承担核心动作，不能只是站在角落。需要多人或家族合影时，为每个角色分别建立身份锚点，并减少单图动作复杂度。

## 消化内容

提炼 4-6 个认知锚点；短文可降为 1-3 个，长文通常不超过 8 个。优先选择：

- 核心判断
- 因果或反馈机制
- 前后对比
- 输入、处理与输出
- 决策分岔
- 容易误解的隐含前提
- 能用物理动作表达的概念隐喻

不要平均配图。每张只解释一个认知动作。

## 先定组图策略

用户只要规划时，输出 shot list：放置位置、主题、核心意思、结构类型、IP 动作、主要物件、短标注。

用户明确说“生成、做图、出图”时，在身份与画风锚点通过后直接生成，不再等待确认。每张图单独调用图像工具；不同主题使用不同提示词，不能用多图拼版代替。

生成前必须读 `references/generation.md` 并遵守 GPT-image-2 硬约束。构图见 `references/composition-patterns.md`，画风见 `references/visual-dna.md`，单图提示词见 `references/prompt-template.md`。

## 一致性规则

- 官方英文角色名是身份主键；中文别名只用于提问和文件名。
- 动画作品英文名与版本是画风主键；不得只写“品牌风”“动漫风”这类模糊标签。
- 每张图复用完全相同的身份不可变量和画风不可变量，只替换动作、道具和构图变量。
- 不依赖 seed 保持一致；以参考图和编辑模式为主。
- 允许动作和表情变化，不允许轮廓、五官数量、毛色分区、服装主特征或材质漂移。
- 若去掉 IP 后隐喻仍完整成立，重写构图，让角色亲手完成关键动作。
- 未经用户要求，不添加第二个 IP、品牌标志或新角色。
- 不混入小黑，不现场创造原创怪角色。

## 检查与保存

按 `references/qa-checklist.md` 逐张检查。身份失败优先改用参考图编辑，不继续文字重抽；中文错字严重时减少标注后重生成。

项目内输出到：

```text
assets/<content-slug>-illustrations/
```

至少保留：

```text
identity-master.png
style-master.png
01-topic.png
02-topic.png
```

报告生成模型、图片数量、所选家族与角色、参考图、两个锚点路径和最终图片路径。
