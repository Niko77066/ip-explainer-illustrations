# 构图与隐喻

每张只选一种结构：

- 流程：一个输入、一个怪装置、一个输出。
- 前后对比：左侧旧状态，右侧新状态，IP 完成中间动作。
- 局部系统：只保留 3-5 个必要部件，IP 操作关键部件。
- 概念隐喻：把抽象概念变成一个低科技物件和一个物理动作。
- 地图路线：一条弯曲路径和少量节点，IP 牵引、修补或守门。
- 小漫画：2-4 格，每格只有一个动作。
- 方法分层：不画正式金字塔，让 IP 搬、叠、拆或支撑不同层。

先把概念改写成动作：筛掉、卡住、变重、回流、分岔、压缩、称量、修补、开门、对齐。

再选 1-2 个物件：纸箱、抽屉、旧机器、漏斗、秤、邮筒、门、井、梯子、水管、线团、转盘、黑盒、打孔器、压面机。

最后让 IP 亲手完成动作。不要照搬已有案例构图，也不要把多个机制塞进同一张图。
