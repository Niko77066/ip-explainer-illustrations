# QA Checklist

## 角色

- IP 与用户选择一致。
- 使用官方英文角色名作为身份主键。
- 精确模式已使用同版本原始参考图、身份锚点和画风锚点。
- 与原始参考和身份锚点一眼可认。
- 没有混入小黑、原创怪角色或其他未选择角色。
- IP 承担核心动作，不是装饰。
- 同组图片的轮廓、五官、主色、服装和 Q 版比例稳定。
- 没有把动画版、毛绒版、手办版或不同服装系列混为一个角色版本。

## 内容

- 一张只解释一个机制。
- 隐喻能在 1 秒内看懂。
- 没有照搬旧案例。
- 中文标注少、短、可读；没有多余标题。

## 画面

- 16:9 横版；动画模式遵循对应背景规则，默认简笔画使用纯白背景。
- 主体不超过约 60%，留白充足。
- 已按角色动画、家族动画、默认简笔画的顺序选择画风。
- 动画模式与对应作品的轮廓、色块、光影、材质和背景密度一致。
- 默认模式以黑色手绘线为主，红橙蓝用色克制。
- 与 `style-master.png` 的线宽、留白、色彩职责和标注密度一致。
- 没有混入同家族其他年代、2D/3D/定格/真人版本。
- 不像 PPT、课程页、商业海报或儿童插画。
- 没有渐变、阴影、纸纹、复杂 UI 或水印。

## 模型与交付

- 生成模型明确记录为 GPT-image-2。
- 没有静默切换到 GPT-image-1.5、DALL-E、Flux、Midjourney 或其他模型。
- 一张图对应一次独立生成调用，没有把整组图拼成一张。
- 最终报告写明模型、图片数量和保存路径。
- 最终报告写明家族、官方英文角色名、版本、参考图与两个锚点路径。

## 失败处理

- 身份漂移：回到原始参考与身份锚点，优先编辑，只改动作；不要继续纯文字重抽。
- 画风错路：回到 `family-style-catalog.md` 重新选择具体作品版本，不用模糊品牌风格词。
- 默认简笔画太可爱或太商业：收紧为冷静手绘草图，去掉装饰、光泽和产品摄影感。
- 文字错误：减少标注；局部编辑仍不稳时重生成。
- 太复杂：删到一个动作、一个主物件和 3-5 个标注。
- IP 只是旁观：让它亲手操作、搬运、修补、称量或推动核心结构。
